# Auction Investment Ranker — 채점 루브릭 & 리포트

경매 물건 다건의 수익성·리스크를 종합해 **투자등급(S/A/B/C/D)** 과 순위를 매기고, 크림+오렌지 톤 HTML 리포트를 생성한다.
디자인 톤이 필요하면 `algo-design` 시스템을 따른다.

## 입력
- `_workspace/04_profit_*.json` (수익성, 필수) — 시나리오별 ROI, 리스크플래그.
- 보강용: `02_rights_*.json`(명도난이도·인수금액), `03_valuation_*.json`(시세신뢰도).

## 채점 루브릭

**1) 정규화** — 전 물건의 기본 시나리오 ROI를 0~1로 min-max 정규화(`roi_norm`).

**2) 리스크점수** (낮을수록 좋음):
| 요소 | 점수 |
|------|------|
| 명도난이도 | 1~5 그대로 |
| 인수과다(ROI 음수 유발) | +3 |
| 시세신뢰도 low | +2 |
| data_gap (권리/시세 결측) | +2 |

`risk_norm` = 리스크점수를 0~1로 정규화.

**3) 종합점수** = `roi_norm × 0.6 − risk_norm × 0.4`.

**4) 등급 컷** (종합점수 내림차순):
- **S**: 상위 10% — 고수익·저리스크
- **A**: ~30%
- **B**: ~60%
- **C**: ~85%
- **D**: 나머지 / 리스크플래그 2개↑는 자동 한 등급 강등

## 출력

### `_workspace/05_ranking.json`
```json
[ { "case":"2024_12345", "등급":"A", "순위":2, "기본ROI":0.18,
    "리스크점수":3, "한줄근거":"저감율 22%·명도난이도 낮음, 시세신뢰도 medium" } ]
```

### `_workspace/05_report.html`
필수 섹션:
1. **요약** — 총 N건 중 분석 M건, 등급 분포(S/A/B/C/D 개수), 미분석 사유.
2. **랭킹 표** — 순위·사건번호·소재지·등급·기본ROI·리스크점수·한줄근거. 등급별 색상(S=진오렌지, D=회색).
3. **상위 물건 카드** (S·A) — 3시나리오 ROI, 총투자금, 인수금액, 명도난이도, 리스크플래그, 예상낙찰가 vs 실거래.
4. **주의 물건** — 리스크플래그 2개↑ / 인수과다 / data_gap 물건을 별도 경고 섹션.
5. **면책** — "실거래·감정가·권리 추정 기반 참고자료이며 투자 판단·손실 책임은 투자자에게 있음."

## 원칙
1. **리스크를 숨기지 않는다.** 고ROI라도 리스크플래그가 크면 강등하고 근거를 노출.
2. **미분석을 감추지 않는다.** data_gap·실패 물건을 반드시 리포트에 명시.
3. 단일 숫자 오도 금지 — 항상 3시나리오(보수/기본/낙관)를 함께 보여준다.
4. 채점 가중치·컷은 상단 상수로 두어 조정 가능하게 한다(과적합 금지).
